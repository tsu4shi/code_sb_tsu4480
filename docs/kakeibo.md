# 家計簿集計ツール（MoneyForward ME）

MoneyForward ME からエクスポートした「収入・支出詳細」CSVを読み込み、明細ごとに負担者（夫 / 妻 / 共通 / 除外）をマークして、月別の出費集計を作る個人利用ツールです。バックエンドやデータベースは使わず、ブラウザ内だけでデータを処理します（CSVはどこにも送信されません）。

**注意:** このツールは家計簿データの集計のみを行います。家計・投資アドバイスや将来予測は行いません。

## 使い方（ブラウザ版・推奨）

```bash
npm install
npm run kakeibo
```

ブラウザで http://localhost:1235/kakeibo.html を開き、以下の手順で操作します。

1. **CSVを読み込む** — MoneyForward ME の「収入・支出詳細」画面からエクスポートしたCSVファイル（`収入・支出詳細_YYYY-MM-DD_YYYY-MM-DD.csv`）を、月をまたいで複数選択します。Shift_JIS（CP932）のまま読み込めます。ドラッグ＆ドロップにも対応しており、複数ファイルをまとめてドロップできます（クリックしてファイル選択ダイアログを開くことも可能）。CSV以外の拡張子のファイルが混ざっていても自動的に無視されます。
2. **一覧を確認する** — 全期間のデータが1つの表にまとまって表示されます。MoneyForward側の ID をキーに重複行は自動的に除外されます。ID自体は画面には表示されませんが、内部的に保持され、エクスポート時にも保持されます。
3. **負担者をマークする** — 各行の「負担者」列で 夫 / 妻 / 共通 / 除外 を選びます。「除外」は銀行振り込みなど口座間の資金移動だけで、MoneyForward側で振替として検出されなかった明細（例:「現金・カード」カテゴリの引き出し等）を集計から手動で外したいときに使います。以下の一括マーク機能も使えます。
   - 「保有金融機関」でまとめてマーク（例: `つよし_PayPayカード` → 夫）
   - 「内容」でまとめてマーク（例: `三井アウトレットパーク入間` → 妻、`振込 ヤマモト` → 除外）— 同じ店舗・同じ摘要の明細をまとめて処理したいときに便利です
   - どちらも「既存のマークを上書きする」チェックボックス（デフォルト: 未チェック）があり、未チェックの場合は既にマーク済みの明細をスキップし、チェックすると既存のマークも含めて一括で上書きします
4. **メモを編集する（任意）** — 各行の「メモ」列は編集可能なテキスト欄です。MoneyForward ME側で入力したメモが最初から表示され、クリックして自由に書き換えられます（Enterキーまたはフォーカスを外すと確定）。
5. **集計表を確認する** — マークをキーに、月別 x 負担者別の出費合計が自動的に更新されます。

「カテゴリ」と「負担者」の間の「タグ」列には、以下のバッジが表示されることがあります（バッジにマウスを乗せると説明が出ます）。

| バッジ | 由来 |
|---|---|
| 振替 | MoneyForward Me自身のデータ（CSVの「振替」列が`1`） |
| 対象外 | MoneyForward Me自身のデータ（CSVの「計算対象」列が`0`、MoneyForward Me側で計算対象外に設定した明細） |
| 収入 | **このツール独自の判定**。MoneyForwardには収入専用の列がないため、金額がプラスであることから自動的に付けています |
| 除外 | **このツール独自のタグ**。負担者を手動で「除外」に設定した明細にのみ表示されます |

## マーク・メモの保存と引き継ぎ

マークとメモの編集内容は、どちらもブラウザの localStorage に保存され、**同じブラウザで同じCSVを再度読み込めば**（MoneyForwardのIDをキーに）自動的に復元されます。別のブラウザ・別端末に引き継ぐ、またはバックアップを取る場合は次のいずれかの方法が使えます。

- **「全データCSVをダウンロード」→ 次回そのCSVだけを読み込む** — 明細データ・マーク・編集後のメモ・IDがすべて1つのCSVに保存されるため、**元のMoneyForward CSVを再度エクスポートし直さなくても**、このCSVを「1. CSVを読み込む」の入力欄に選択するだけで一覧・マーク・メモ・集計表がすべて復元されます（新しい月の生CSVと一緒に選んでもOKです。IDで重複排除されます）。
- **「マークをエクスポート」→「マーク・メモをインポート」** — マークだけをJSONファイルとして保存・復元する場合はこちら（メモの保存はJSON形式には含まれません。メモも一緒に保存したい場合は「全データCSVをダウンロード」を使ってください）。この場合は元のMoneyForward CSVを毎回読み込み直す必要があります。
- 「マーク・メモをインポート」ボタンは、JSON（マークのみ）と 全データCSV（マーク+メモ両方）のどちらも自動判別して受け付けます。全データCSVを渡した場合はメモも復元されます。

## 公開してスマホからも使う（GitHub Pages）

このツールは完全にクライアントサイド（ブラウザ内だけ）で動くため、サーバーを用意しなくても静的サイトとしてホスティングでき、スマホのブラウザからもアクセスできるようになります。このリポジトリには GitHub Pages への自動デプロイ用ワークフロー（`.github/workflows/deploy-pages.yml`）が含まれています。

**有効化手順（初回のみ、GitHub上での手動操作が必要）:**

1. GitHubのリポジトリ設定 → **Settings → Pages** を開く
2. **Source** を **GitHub Actions** に設定する
3. `main` ブランチにpushすると自動的にビルド・デプロイされます（`kakeibo.html` や `src/kakeibo/**` を変更した場合）。手動で実行したい場合は **Actions** タブから「Deploy to GitHub Pages」ワークフローを **Run workflow** で起動できます。
4. デプロイ後は `https://<GitHubユーザー名>.github.io/<リポジトリ名>/kakeibo.html` でアクセスできます（例: `https://tsu4shi.github.io/code_sb_tsu4480/kakeibo.html`）。このURLをスマホのブラウザで開けばそのまま使えます。

**注意点:**

- **このリポジトリはパブリック（公開）リポジトリのため、デプロイされたページも認証なしで誰でも開けます。** ただしCSVを自分でアップロードしない限り何のデータも表示されず、処理はすべてブラウザ内で完結し外部（サーバーを含む）へは一切送信されないため、あなたの家計簿データそのものが漏れることはありません。とはいえURLを知っていれば誰でもこのツール自体は開けてしまう点はご留意ください。より厳密にアクセス制限したい場合は、リポジトリをプライベートにした上でGitHub Pro/Team等のプライベートリポジトリ向けPages機能を使うか、Cloudflare Pages + Cloudflare Access のような認証つきの静的ホスティングサービスを検討してください。
- ブラウザのlocalStorageはオリジン（ドメイン+パス）ごとに独立しているため、ローカルで `npm run kakeibo`（`http://localhost:1235`）を使ったときのマーク・メモと、GitHub Pages版（`https://.../kakeibo.html`）のマーク・メモは別々に保存されます。両方を行き来したい場合は「全データCSVをダウンロード」→ 別環境で読み込む、という形でデータを持ち運んでください。

## 使い方（CLI版・スプレッドシート派向け）

ブラウザを使わず、CSVを1つにまとめてExcel/Googleスプレッドシートでマークしたい場合は次のCLIを使います。

```bash
# 1. 生データのCSVを1つのディレクトリに集める（Gitには絶対にコミットしないこと）
mkdir -p data/moneyforward
cp ~/Downloads/収入・支出詳細_*.csv data/moneyforward/

# 2. 1つのCSVにまとめる（負担者列は空欄）
node tools/kakeibo/cli.js combine --input-dir data/moneyforward --out data/kakeibo-ledger.csv

# 3. Excel等で開き、「負担者」列に 夫 / 妻 / 共通 / 除外 のいずれかを入力して保存する

# 4. マーク済みCSVから月別集計を作る
node tools/kakeibo/cli.js summarize --marks data/kakeibo-ledger.csv
```

`summarize` は月別 x 負担者別の集計を JSON で標準出力します。

## 集計ロジック

- 収入行・振替行（口座間振替、`振替`列が `1`）・MoneyForward側で「計算対象外」に設定した行（`計算対象`列が `0`）は、出費集計から**自動的に**除外されます。
- 集計対象になるのは「支出」（金額がマイナス）かつ計算対象かつ非振替の行のみです。
- 負担者を **「除外」に手動でマーク**した明細は、集計から完全に除かれます（月別集計表の「未設定」にも「夫/妻/共通」にも計上されません）。銀行振り込みなど、MoneyForward側で振替として検出されなかった口座間の資金移動をなかったことにしたい場合に使います。除外した件数・金額は集計表の下の注記に表示されます。
- 負担者が未設定の支出は集計表の「未設定」列にまとめて表示され、合計には含まれますが 夫/妻/共通 のどれにも計上されません。マーク漏れの確認に使えます。

## データ形式（MoneyForward ME エクスポートCSV）

| 列名 | 内容 |
|---|---|
| 計算対象 | `1`=集計に含める, `0`=除外 |
| 日付 | `YYYY/MM/DD` |
| 内容 | 明細の摘要 |
| 金額（円） | マイナス=支出、プラス=収入 |
| 保有金融機関 | 口座・カード名（例: `つよし_PayPayカード`） |
| 大項目 / 中項目 | MoneyForwardのカテゴリ |
| メモ | 任意メモ |
| 振替 | `1`=口座間振替（集計除外）, `0`=通常明細 |
| ID | MoneyForward側で発行される行の一意なID（重複排除・マーク/メモの永続化キーに使用）。画面には表示されませんが内部的に保持され、エクスポート時にも保持されます |

エンコーディングはShift_JIS（CP932）です。列の並び順が変わっていても列名で自動的に対応します。

## プライバシー

- このリポジトリには実際の家計簿データ（CSV・エクスポート結果）を **絶対にコミットしないでください**。`data/` ディレクトリは `.gitignore` 済みです。
- ブラウザ版はCSVをサーバーに送信せず、すべてクライアントサイド（JavaScript）で処理します。
- CLI版は完全にローカルで完結し、外部通信は行いません。

## プロジェクト構成

```
src/kakeibo/
  csv.js                    汎用CSVパーサ（引用符・改行対応）
  parseMoneyForwardCsv.js    MoneyForward ME CSV → トランザクション配列
  combineTransactions.js     複数ファイルの結合・重複排除
  aggregate.js               月別 x 負担者別の集計ロジック・負担者ラベル定義
  ledgerCsv.js               自前の「全データCSV」形式の書き出し/読み込み（マーク・メモ込み）
  memoOverrides.js           メモ編集内容を再読み込み時に復元するロジック
  kakeiboApp.js               ブラウザUI（file input, 一括マーク, メモ編集, 集計表表示）
  kakeiboStyles.css           ブラウザUIのスタイル
tools/kakeibo/cli.js          CLI（combine / summarize サブコマンド）
test/kakeibo/kakeibo.test.js  ユニットテスト（架空データのみ使用）
kakeibo.html                  ブラウザUIのエントリーポイント
```
